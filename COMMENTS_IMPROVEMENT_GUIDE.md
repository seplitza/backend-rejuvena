# Руководство по обновлению системы комментариев

## Обзор изменений

Реализованы следующие улучшения для системы комментариев в админ-панели:

### 1. Вкладка "Ответы админа"
- Добавлена отдельная вкладка для просмотра всех ответов администраторов
- Ответы админа отделены от общего потока модерации
- Счетчик ответов админа выведен в отдельную статистику
- Ответы админа **не включаются** в общий счетчик "На модерации"

### 2. Фильтр по упражнениям
- Добавлен выпадающий список упражнений с количеством комментариев
- Фильтр работает на обеих вкладках
- Автоматическая загрузка списка упражнений при открытии страницы

### 3. Звездочка для важных вопросов
- Возможность пометить комментарий как "важный" (starred)
- Звездочка отображается в списке комментариев (⭐)
- Кнопка переключения в модальном окне детального просмотра
- **Важные комментарии показываются в топе** для пользователей в приложении

### 4. Сортировка для пользователей
В мобильном/веб приложении комментарии сортируются в следующем порядке:
1. **Звездные комментарии** (важные, отмеченные админом)
2. **Личные ответы админа текущему пользователю**
3. Все остальные по дате (новые первые)

## Изменения в базе данных

### Модель Comment
Добавлено новое поле:
```typescript
starred: boolean // По умолчанию false
```

### Индексы
Добавлен индекс для поля `starred` для быстрой выборки важных комментариев.

## Изменения в API

### 1. GET `/api/admin/comments`
**Новые параметры:**
- `view` (query): `'moderation'` | `'admin-replies'` - переключение между вкладками
- `exerciseId` (query): фильтр по упражнению

**Обновленный ответ:**
```json
{
  "success": true,
  "comments": [...],
  "stats": {
    "pending": 2,
    "urgent": 1,
    "needsResponse": 10,
    "adminReplies": 21  // НОВОЕ
  }
}
```

### 2. PUT `/api/admin/comments/:id/starred`
**Новый endpoint для переключения звездочки**
```typescript
Request: { "starred": true | false }
Response: { "success": true, "data": Comment }
```

### 3. GET `/api/admin/comments/exercises/list`
**Новый endpoint для получения списка упражнений с комментариями**
```typescript
Response: {
  "success": true,
  "exercises": [
    { "_id": "...", "title": "Название упражнения", "count": 15 }
  ]
}
```

### 4. GET `/api/comments` (пользовательский)
**Обновлена сортировка:**
- Starred comments → Admin replies to user → All others (by date)

## Особенности реализации

### Фильтрация "Ответы админа"
Backend фильтрует комментарии следующим образом:
- **Общий поток**: Корневые комментарии (без parent) + ответы пользователей без ответа админа
- **Ответы админа**: Только комментарии, где есть `parentCommentId` и `respondedBy`

### Сортировка в приложении
Реализована custom сортировка в `/api/comments`:
1. Сначала все комментарии с `starred: true`
2. Затем комментарии текущего пользователя, на которые есть ответ админа
3. Остальные по дате (новые первые)

### Производительность
Для оптимизации добавлены индексы:
- `starred: 1` - для быстрой выборки важных комментариев
- Существующие индексы по `exerciseId`, `status`, `parentCommentId` и т.д.

## Troubleshooting

### Вкладка "Ответы админа" пустая
- Проверьте, что есть комментарии с заполненным полем `respondedBy`
- Убедитесь, что параметр `view=admin-replies` передается корректно

### Фильтр по упражнениям не работает
- Проверьте endpoint `/api/admin/comments/exercises/list`
- Убедитесь, что в БД есть комментарии с `exerciseId`

### Звездочка не сохраняется
- Проверьте endpoint `/api/admin/comments/:id/starred`
- Убедитесь, что поле `starred` добавлено в модель Comment

### Сортировка не работает в приложении
- Проверьте endpoint `/api/comments`
- Убедитесь, что обновленная логика сортировки развернута
